# Arvore

## O que é uma árvore?

Uma árvore é uma estrutura de dados hierárquica que consiste em nós conectados por arestas. Cada nó pode ter um ou mais filhos, mas apenas um pai (exceto o nó raiz, que não tem pai). As árvores são usadas para representar relações hierárquicas, como a estrutura de diretórios em um sistema de arquivos, a organização de uma empresa ou a estrutura de uma expressão matemática.

## Terminologia

- Nó: Um elemento da árvore que contém um valor ou dados.
- Raiz: O nó superior da árvore, que não tem pai.
- Folha: Um nó que não tem filhos.
- Subárvore: Uma parte da árvore que é composta por um nó e todos os seus descendentes.
- Altura: O número de arestas no caminho mais longo da raiz até uma folha.
- Profundidade: O número de arestas do nó até a raiz.
- Grau: O número de filhos de um nó.
- Caminho: Uma sequência de nós e arestas conectados.

## Tipos de árvores

- Árvore binária: Cada nó tem no máximo dois filhos.
- Árvore binária de busca: Uma árvore binária onde os nós são organizados de forma que o valor do nó à esquerda seja menor que o valor do nó pai, e o valor do nó à direita seja maior.
- Árvore AVL: Uma árvore binária de busca auto-balanceada onde a diferença de
altura entre as subárvores de qualquer nó é no máximo 1.
- Árvore B: Uma árvore auto-balanceada que mantém os dados ordenados e permite buscas, inserções e exclusões eficientes.
- Árvore Trie: Uma árvore usada para armazenar um conjunto de strings, onde cada nó representa um caractere.

## Operações em árvores

- Inserção: Adicionar um novo nó à árvore.
- Busca: Encontrar um nó com um valor específico.
- Remoção: Excluir um nó da árvore.
- Travessia: Visitar todos os nós da árvore em uma ordem específica (pré-ordem, em-ordem, pós-ordem).
- Balanceamento: Ajustar a estrutura da árvore para manter um desempenho eficiente.

## Aplicações de árvores

- Estruturas de diretórios em sistemas de arquivos.
- Representação de expressões matemáticas.
- Organização hierárquica de dados, como em bancos de dados.
- Algoritmos de busca e ordenação.
- Implementação de tabelas de símbolos em compiladores.
- Estruturas de dados para jogos, como árvores de decisão.
- Redes de computadores, como árvores de roteamento.
- Inteligência artificial, como árvores de decisão e árvores de busca.
- Estruturas de dados para armazenamento e recuperação de dados, como árvores B e árvores AVL.
- Estruturas de dados para indexação, como árvores Trie.
